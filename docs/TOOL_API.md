# HAL YieldOS — Tool API Reference

The `YieldOSToolAPI` class provides structured, token-efficient access to YieldOS
case output objects. It is designed to be called by AI agents or orchestrators that
need to reason over case results without reading raw industrial logs.

---

## Evidence Compression Ratio (ECR)

**Evidence Compression Ratio** = raw input tokens / YieldOS API summary tokens

Raw industrial log files are large. The YieldOS Tool API surfaces only the
structured objects that matter for decision-making, achieving a compression ratio
of 10x–45x depending on the domain.

| Domain | Raw input tokens (est.) | API summary tokens (est.) | ECR | Note |
|--------|------------------------:|-------------------------:|----:|------|
| Semiconductor fab | 30,690 | 680 | **45x** | 500-row synthetic tool log |
| Robot telemetry | 7,987 | 736 | **11x** | 500-row synthetic telemetry |
| Space telemetry | 8,810 | 733 | **12x** | 500-row synthetic telemetry |
| **Total / average** | **47,487** | **2,149** | **22x** | — |

> **sample-based estimate**: All token counts are derived from 500-row synthetic
> datasets generated by `yieldos <domain> gen`. Actual compression ratios vary
> with dataset size — larger raw logs produce higher ECR values. Counts are
> estimated using the cl100k_base tokenizer.

---

## Installation

```python
from yieldos.api.tool_api import YieldOSToolAPI
```

No additional installation needed — `YieldOSToolAPI` uses only Python stdlib.
Run a domain analysis first to produce the case output directory.

---

## Quick Start

```python
from yieldos.api.tool_api import YieldOSToolAPI

# Point to any completed case output directory
api = YieldOSToolAPI("output/semfab_case_001")

# Most token-efficient: single call returns everything
summary = api.get_full_summary()

# Or call individual accessors
state   = api.get_state_snapshot()
pack    = api.get_evidence_pack()
ooda    = api.get_ooda_frame()
rca     = api.get_root_cause_candidates()
rec     = api.get_recovery_candidates()
missing = api.get_missing_evidence_request()
```

## API Reference

### `YieldOSToolAPI(case_dir: str)`

Constructor. `case_dir` must be a directory produced by a YieldOS domain analysis.

```python
api = YieldOSToolAPI("output/semfab_case_001")
```

---

### `get_full_summary() -> dict`

**Recommended for AI agents.** Returns a single compact dict containing all
output objects an agent needs to reason over the case. This is the most
token-efficient call.

```python
summary = api.get_full_summary()
# Returns:
{
  "state": {
    "case_id": "semfab_case_001",
    "domain": "semfab",
    "asset_id": "TOOL_STEP04",
    "state": "fault_candidate",
    "severity": "high",
    "confidence": 0.87,
    "mode": "read_only_shadow",
    "metrics": {"drift_sigma": 3.8, "cd_shift_nm": 2.1, "yield_loss_pct": 6.5},
    "created_at": "2025-01-15T09:30:00.123456+00:00"
  },
  "ooda": {
    "case_id": "semfab_case_001",
    "observe": "Tool drift detected in STEP_04: pressure_torr at 3.8 sigma",
    "orient": "Cross-step correlation: drift -> CD shift -> yield loss",
    "decide": "Request engineer review of STEP_04 pressure control history",
    "act": "recommendation_only_no_hardware_action"
  },
  "top_candidates": [
    {"candidate": "pressure_torr_drift", "confidence": 0.87, "claim_boundary": "candidate_only"}
  ],
  "top_recovery": [
    {"action": "recommend_inspection", "risk": "low", "execution_mode": "recommendation_only",
     "hardware_execution_enabled": false, "requires_human_review": true}
  ],
  "missing_evidence": [
    {"item": "maintenance_log", "reason": "Cannot confirm drift cause without PM history", "priority": "high"}
  ]
}
```

### `get_state_snapshot() -> dict`

Returns the `StateSnapshot` fields. `~50 tokens` vs thousands for raw logs.

```python
state = api.get_state_snapshot()
```

| Field | Type | Description |
|-------|------|-------------|
| `case_id` | str | Unique case identifier |
| `domain` | str | `robot` / `space` / `semiconductor` / `semiforge` |
| `asset_id` | str | The asset under analysis |
| `state` | str | `nominal` / `degraded` / `fault_candidate` / `under_investigation` |
| `severity` | str | `info` / `low` / `medium` / `high` / `critical` |
| `confidence` | float | 0.0 – 1.0 |
| `mode` | str | Always `read_only_shadow` |
| `metrics` | dict | Domain-specific numeric metrics |
| `created_at` | str | ISO 8601 timestamp |

---

### `get_evidence_pack(include_objects: bool = True) -> dict`

Returns the `EvidencePack` fields. The `checksum` can be used to verify
the pack was not modified since sealing.

```python
pack = api.get_evidence_pack()
pack_slim = api.get_evidence_pack(include_objects=False)  # omit evidence_objects list
```

| Field | Type | Description |
|-------|------|-------------|
| `case_id` | str | Case identifier |
| `domain` | str | Domain |
| `summary` | str | Human-readable case summary |
| `causal_claim_boundary` | str | Always `candidate_only_not_certified_cause` |
| `root_cause_candidates` | list | Ranked RCA candidates |
| `missing_evidence` | list | Items needed to close the investigation |
| `checksum` | str | `sha256:` + hex; verifiable with `yieldos validate` |
| `evidence_objects` | list | (if `include_objects=True`) Evidence items sorted by confidence |

**Evidence object fields** (when `include_objects=True`):

| Field | Description |
|-------|-------------|
| `id` | `evidence_id` |
| `type` | `tool_drift` / `wafer_map` / `metrology_shift` / `fault_code` / `threshold_breach` / etc. |
| `summary` | One-line description |
| `confidence` | 0.0 – 1.0 |

---

### `get_ooda_frame() -> dict`

Returns the OODA loop fields.

```python
ooda = api.get_ooda_frame()
```

| Field | Description |
|-------|-------------|
| `observe` | Raw observations extracted from data |
| `orient` | Interpretation and working hypothesis |
| `decide` | Recommended human action |
| `act` | Always `recommendation_only_no_hardware_action` |

---

### `get_root_cause_candidates() -> list`

Returns the list of `RootCauseCandidate` dicts from the `EvidencePack`.

```python
candidates = api.get_root_cause_candidates()
for c in candidates:
    print(c["candidate"], c["confidence"], c["claim_boundary"])
```

| Field | Description |
|-------|-------------|
| `candidate` | Hypothesis description |
| `confidence` | 0.0 – 1.0 |
| `claim_boundary` | Always `candidate_only` |
| `supporting_evidence` | List of evidence IDs |

---

### `get_recovery_candidates() -> list`

Returns the list of `RecoveryCandidate` dicts. All candidates have
`hardware_execution_enabled = false` and `requires_human_review = true`.

```python
recovery = api.get_recovery_candidates()
for r in recovery:
    print(r["action"], r["risk"], r["execution_mode"])
```

| Field | Value |
|-------|-------|
| `action` | One of the permitted action names (see SAFETY_BOUNDARY.md) |
| `expected_benefit` | str |
| `risk` | `low` / `medium` / `high` |
| `execution_mode` | `recommendation_only` or `human_review_required` |
| `hardware_execution_enabled` | Always `false` |
| `requires_human_review` | Always `true` |

---

### `get_missing_evidence_request() -> list`

Returns the `missing_evidence` list from the `EvidencePack`.

```python
missing = api.get_missing_evidence_request()
for m in missing:
    print(m["item"], m["priority"], m["reason"])
```

| Field | Description |
|-------|-------------|
| `item` | Name of the missing evidence item |
| `reason` | Why it is needed |
| `priority` | `high` / `medium` / `low` |

---

### `get_experience_records(experience_path: str) -> list`

Loads records from an `ExperienceGraph` JSONL ledger.

```python
records = api.get_experience_records("output/experiences.jsonl")
for r in records:
    print(r["domain"], r["selected_action"], r["delta"])
```

## Verifying a Case

The `EvidencePack` checksum covers all 11 content fields:

```python
import json, hashlib

pack = json.loads(open("output/semfab_case_001/evidence_pack.json").read())
payload = {k: pack[k] for k in (
    "schema", "case_id", "domain", "asset_id", "summary",
    "causal_claim_boundary", "evidence_objects", "root_cause_candidates",
    "missing_evidence", "state_snapshot_ref", "created_at",
) if k in pack}
blob = json.dumps(payload, sort_keys=True, separators=(",", ":")).encode("utf-8")
expected = "sha256:" + hashlib.sha256(blob).hexdigest()
assert pack["checksum"] == expected, "Checksum mismatch!"
```

Or via CLI:

```bash
yieldos validate --case output/semfab_case_001
```

## Output File Reference

Each domain analysis produces these files in the output directory:

| File | Loaded by |
|------|-----------|
| `state_snapshot.json` | `get_state_snapshot()` |
| `evidence_pack.json` | `get_evidence_pack()`, `get_root_cause_candidates()`, `get_missing_evidence_request()` |
| `ooda_frame.json` | `get_ooda_frame()` |
| `recovery_candidates.json` | `get_recovery_candidates()` |
| `report.md` | Human reading |
| `report.html` | Browser / stakeholder sharing |

## Safety Constraints

The Tool API is read-only. It reads existing output files only.
It never writes, modifies, or transmits data to external systems.

All data returned by the API carries the same safety invariants as the objects
it reads:

```
hardware_execution_enabled = false
causal_claim_boundary      = candidate_only_not_certified_cause
execution_mode             = recommendation_only | human_review_required
mode                       = read_only_shadow
act                        = recommendation_only_no_hardware_action
```

See [SAFETY_BOUNDARY.md](SAFETY_BOUNDARY.md) for the complete specification.
