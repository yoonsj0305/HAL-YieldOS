# GitHub Upload Checklist

HAL YieldOS v3.0.11

---

## Before creating repository

- [ ] Choose repository name (recommended: `HAL-YieldOS`)
- [ ] Confirm public visibility
- [ ] Confirm license (Evaluation Only — see LICENSE.txt)
- [ ] Confirm no private data in any committed file
- [ ] Confirm `output/` folder is not committed (check .gitignore)
- [ ] Confirm `dist/` folder is not committed
- [ ] Confirm no generated `.pyc` or `__pycache__/` directories committed
- [ ] Confirm no real industrial telemetry data is present
- [ ] Run `python scripts/check_launch_guard.py` — must PASS

## After initial push

- [ ] Check README rendering on GitHub
- [ ] Check badges display correctly (PoC, read-only, candidate-only, CI badge)
- [ ] Verify GitHub Actions tab shows the `tests` workflow
- [ ] Run `tests` workflow manually once to confirm it passes
- [ ] Confirm launch guard passes on GitHub Actions ubuntu-latest (cross-platform path fix verified in v3.0.10)
- [ ] Run `public-demo` workflow manually to confirm demo passes
- [ ] Confirm no `recovery_profile.json` was generated in CI artifacts
- [ ] Set repository About description (see docs/GITHUB_REPO_METADATA.md)
- [ ] Add repository topics (see docs/GITHUB_REPO_METADATA.md)
- [ ] Inspect all documentation links in README

## Safety boundary review

- [ ] README states YieldOS is read-only
- [ ] README states outputs are candidate-only
- [ ] README states human review is required
- [ ] README states no hardware control
- [ ] README states no recipe control
- [ ] README states no yield guarantee
- [ ] README links to docs/PUBLIC_SAFETY_BOUNDARY.md

## Create first release

- [ ] Create tag `v3.0.11`
- [ ] Create GitHub release titled:
  ```
  HAL YieldOS v3.0.11 - Release Artifact Hygiene Enforcement + Dirty Archive Prevention Patch
  ```
- [ ] Attach release asset:
  ```
  HAL-YieldOS-v3.0.11-poc-release.zip
  ```
- [ ] Copy release body from docs/GITHUB_LAUNCH_NOTES.md and RELEASE_NOTES.md
- [ ] Confirm release asset does not include `output/`, `dist/`, or `__pycache__/`

## Release artifact rule

Only upload the zip generated by:

```bash
python scripts/build_release.py
```

Upload this file:

```
dist/HAL-YieldOS-v3.0.11-poc-release.zip
```

Do not manually compress the working directory.

Do not upload a zip that contains:

- `build/`
- `dist/`
- `output/`
- `outputs/`
- `.ruff_cache/`
- `.pytest_cache/`
- `dist_v*/`
- `__pycache__/`
- `*.pyc`
- `*.whl`
- `*.tar.gz`
- nested `*.zip`

Before uploading, run:

```bash
python scripts/check_release_artifact.py dist/HAL-YieldOS-v3.0.11-poc-release.zip
```

Expected:

```
HAL YieldOS release artifact hygiene: PASS
```
