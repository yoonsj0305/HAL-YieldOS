# HAL YieldOS — GitHub Release Checklist

Use this checklist before tagging and publishing each release.

---

## 1. Core Test Suite

```bash
python -m pytest -q
python -m pytest -q -m cli_e2e
python -m pytest -q -m release_heavy
python -m pytest -q -m installed_wheel
python -m pytest -q -m packaging
```

All must exit normally with 0 failures.

---

## 2. Code Quality (if ruff available)

```bash
python -m ruff check .
```

---

## 3. Health and Demo

```bash
yieldos doctor --deep

yieldos demo --all --out output/demo_all

yieldos validate --case output/demo_all/robot --strict
yieldos validate --case output/demo_all/space --strict
yieldos validate --case output/demo_all/semiconductor --strict
yieldos validate --case output/demo_all/semiforge --strict
yieldos validate --case output/demo_all/memory --strict
```

---

## 4. Semiconductor Pilot Pack

```bash
yieldos semiconductor pilot-pack \
  --input samples/pilot_semiconductor \
  --out output/semiconductor_pilot_pack

yieldos validate --case output/semiconductor_pilot_pack --strict
```

Confirm these files exist:

- [ ] `functional_passport.json` — has `semiconductor_pilot_context`
- [ ] `functional_passport.json` — has `confidence_explanation`
- [ ] `semiconductor_confidence_report.json` — has `missing_metrics`, `available_metrics_summary`
- [ ] `semiconductor_recovery_compiler_export.json` — `recovery_profile_generated = false`
- [ ] `semiconductor_handoff_manifest.json` — `forbidden_files` includes `recovery_profile.json`
- [ ] `report.html` — includes Semiconductor Process Confidence section
- [ ] `recovery_profile.json` — **must NOT exist**

---

## 5. Robot Pilot Pack

```bash
yieldos robot pilot-pack \
  --input samples/pilot_robot \
  --out output/robot_pilot_pack

yieldos validate --case output/robot_pilot_pack --strict
```

## 6. Public Demo Script

```bash
python scripts/run_public_demo.py
```

All steps must PASS.

---

## 7. FYFab Demo

```bash
yieldos semiforge fyfab-demo --out output/fyfab_seed
yieldos validate --case output/fyfab_seed --strict
```

---

## 8. Safety Invariant Check

Confirm across outputs:

- [ ] No `hardware_control_enabled: true` anywhere
- [ ] No `recipe_control_enabled: true` anywhere
- [ ] No `timing_closure_certified` claim anywhere
- [ ] No `yield_guarantee` claim anywhere
- [ ] No `certified_root_cause` claim anywhere
- [ ] `recovery_profile.json` not generated by any flow
- [ ] All `RecoveryCandidate` objects have `hardware_execution_enabled: false`

---

## 9. Release Build

```bash
python scripts/build_release.py
```

Expected output:

```
dist/HAL-YieldOS-v3.0.11-poc-release.zip
```

Internal root folder:

```
HAL-YieldOS-v3.0.11-poc-release/
```

The ZIP **must NOT include**:

- `build/`
- `dist/`
- `output/`
- `.pytest_cache/`
- `.ruff_cache/`
- `__pycache__/`
- `*.pyc`
- `*.whl`
- `*.tar.gz`
- nested `*.zip`
- `hal_yieldos.egg-info/`

---

## 10. Artifact Hygiene Check

After building, verify the release zip is clean:

```bash
python scripts/check_release_artifact.py dist/HAL-YieldOS-v3.0.11-poc-release.zip
```

Expected:

```
HAL YieldOS release artifact hygiene: PASS
```

Do not upload:

- manually zipped working directory
- zip containing `build/`, `dist/`, `output/`, `.ruff_cache/`, `dist_v*/`, `__pycache__/`, `*.pyc`, `*.whl`, `*.tar.gz`
- nested `*.zip` inside the release archive

---

## 11. Installed Wheel Validation (optional)

```bash
python -m build

python -m venv /tmp/yieldos-wheel-test
source /tmp/yieldos-wheel-test/bin/activate
pip install dist/*.whl

yieldos version
yieldos doctor --deep
yieldos demo --all --out /tmp/yieldos-wheel-demo
yieldos validate --case /tmp/yieldos-wheel-demo/semiconductor --strict
```

---

## 11. Git Tag

```bash
git tag v3.0.11
git push origin v3.0.11
```

Do **not** commit generated `output/` or `dist/` folders.

---

## 12. Documentation Check

Confirm public docs do not contain:

- [ ] "safety certification" implied as YieldOS output
- [ ] "yield guarantee" implied as YieldOS capability
- [ ] "timing closure" implied as YieldOS feature
- [ ] "hardware control" implied as YieldOS behavior
- [ ] "recovery_profile.json generated" without the "not" qualifier
