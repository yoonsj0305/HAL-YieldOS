HAL YieldOS v3.0.7

---

## Prerequisites

```bash
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate
pip install -e .
yieldos doctor --deep
```

## 1. Full 5-Domain Demo

Run all five domain analyzers on bundled sample data:

```bash
yieldos demo --all --out output/demo_all
```

Expected domains generated:

- `output/demo_all/robot/`
- `output/demo_all/space/`
- `output/demo_all/semiconductor/`
- `output/demo_all/semiforge/`
- `output/demo_all/memory/`

Validate all:

```bash
yieldos validate --case output/demo_all/robot --strict
yieldos validate --case output/demo_all/space --strict
yieldos validate --case output/demo_all/semiconductor --strict
yieldos validate --case output/demo_all/semiforge --strict
yieldos validate --case output/demo_all/memory --strict
```

Or using the public demo script:

```bash
python scripts/run_public_demo.py
```

## 2. Semiconductor Pilot-Pack Demo

The semiconductor pilot-pack generates a full Recovery Compiler intake bundle:

```bash
yieldos semiconductor pilot-pack \
  --input samples/pilot_semiconductor \
  --out output/semiconductor_pilot_pack

yieldos validate --case output/semiconductor_pilot_pack --strict
```

Key files to inspect:

| File | What it shows |
|------|---------------|
| `functional_passport.json` | `confidence_explanation`, `semiconductor_pilot_context`, remaining/blocked roles |
| `semiconductor_evidence_completeness_report.json` | Evidence coverage per domain area |
| `semiconductor_functional_region_map.json` | Candidate functional die regions |
| `semiconductor_role_candidate_map.json` | Remaining vs. blocked role candidates |
| `semiconductor_valid_conditions_report.json` | Valid operating condition candidates |
| `semiconductor_confidence_report.json` | Process confidence score, missing watched metrics |
| `semiconductor_recovery_compiler_export.json` | Candidate-only export for offline Recovery Compiler testing |
| `semiconductor_handoff_manifest.json` | Authorized handoff file set |
| `report.html` | Human-readable report with confidence section |

> **Important:** `semiconductor_recovery_compiler_export.json` is a candidate-only export artifact for offline testing. It is **not** a recovery profile. It does not trigger Recovery Compiler execution. `recovery_profile.json` is never generated by YieldOS.

## 4. SemFab Confidence Explanation Demo

The confidence report shows which watched metrics have data and which are missing:

```bash
# Run the semiconductor demo and inspect the confidence report
yieldos semifab analyze \
  --input samples/pilot_semiconductor \
  --out output/semfab_confidence_demo \
  --asset chip_demo

# Check the confidence report
cat output/semfab_confidence_demo/semiconductor_confidence_report.json
```

Look for in `confidence_report`:

```json
{
  "missing_metrics": ["gas_flow_sccm", "endpoint_signal"],
  "available_metrics_summary": {
    "summary_text": "1/3 available metrics show drift (pressure_mTorr); 2 watched metrics have no data (gas_flow_sccm, endpoint_signal)",
    ...
  }
}
```

And in `functional_passport.json`:

```json
{
  "confidence_explanation": {
    "missing_metrics": ["gas_flow_sccm", "endpoint_signal"],
    "missing_metric_messages": [
      "gas_flow_sccm: no data",
      "endpoint_signal: no data"
    ],
    ...
  }
}
```

The `report.html` also contains the Semiconductor Process Confidence section with a per-metric table.

## 5. Memory Product Demo

Demonstrates that a device binary policy would discard actually contains recoverable functional capacity:

```bash
yieldos memory product-demo --out output/product_memory_rebinning_demo

cat output/product_memory_rebinning_demo/baseline_vs_yieldos.json
```

Expected result:

```json
{
  "binary_policy_verdict": "FAIL",
  "reclassification_occurred": true,
  "recovered_functional_capacity_estimate": {
    "safe_gb": 22.0,
    "approximate_cache_gb": 4.0,
    "read_only_archive_gb": 3.0,
    "discard_gb": 3.0
  }
}
```

## What Not to Demo

To preserve the YieldOS safety boundary:

- **Do not** show hardware control output
- **Do not** imply recipe changes are made
- **Do not** present outputs as safety certification
- **Do not** imply yield is guaranteed
- **Do not** call Recovery Compiler from YieldOS
- **Do not** generate `recovery_profile.json` — YieldOS never generates it
